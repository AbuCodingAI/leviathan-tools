DEPLOYMENT GUIDE:
-----------------

1. Copy binaries to system:
   sudo cp {scribe,tabula,swipe,appbase}/scribe /usr/local/bin/
   sudo cp {scribe,tabula,swipe,appbase}/tabula /usr/local/bin/
   sudo cp {scribe,tabula,swipe,appbase}/swipe /usr/local/bin/
   sudo cp {scribe,tabula,swipe,appbase}/appbase /usr/local/bin/

2. Make executable:
   chmod +x /usr/local/bin/{scribe,tabula,swipe,appbase}

3. Create desktop files for launcher (in ~/.local/share/applications/):
   [Desktop Entry]
   Name=Scribe
   Type=Application
   Exec=scribe %F
   Icon=document-text
   Categories=Office;WordProcessor;
   MimeType=text/plain;

4. Optional: Register with system
   update-mime-database ~/.local/share/mime
   update-desktop-database ~/.local/share/applications/

KNOWN LIMITATIONS:
------------------

Current Version (Pv3.2):
- No .docx/.xlsx/.pptx support (planned for Pv3.3)
- No formulas in Tabula (planned for Pv4)
- No rich text formatting (basic text only)
- No animations in Swipe (planned)
- SQL support limited to SELECT queries (INSERT/UPDATE planned)
- No network features (planned for future)

Workarounds:
- Export data to plain text or JSON
- Use external tools for format conversion
- Copy/paste between applications

TECHNICAL NOTES:
----------------

1. All applications use GTK3 (not GTK4) for compatibility
2. WebKitGTK 4.1 provides modern web rendering
3. SQLite3 is embedded database with no server
4. All resources (HTML/CSS/JS) are inlined in C code
5. No dynamic memory allocation for main data structures
6. Stack-based allocation with fixed buffer sizes

TROUBLESHOOTING:
----------------

Issue: Application won't start
Solution: Check GTK3 and WebKitGTK 4.1 are installed:
  pkg-config --list-all | grep webkit2gtk

Issue: Slow performance
Solution: This is expected on slow floppy systems:
  - Reduce window size for faster rendering
  - Keep data small (< 10MB)
  - Use SSD/USB instead of floppy for testing

Issue: Memory errors
Solution: Check system has minimum 512 MB RAM:
  free -h
